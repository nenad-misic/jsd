Generating content


You can use DGDL files to generate entities:

  • Create a file with the extension ‘.dgdl'
  • Declare your applications, entities, fields and relationships
  • To create the application run python dgenerator.py my_file.dgdl


Generating applications


As we’ve seen in the previous example, generating applications is quite straightforward, let’s take the previous example and add more things to it:

application appl {
    config {
        frontend react
        server express
        database sqlite
        frontendPort 3000
        serverPort 1337
        authentication jwt
    }
}

Let’s break it down:

  • application is the keyword to say you want to declare an application, followed by application name: appl
  • config to say that you want to specify the configuration
    • we’ll later see that you can also declare entities in applications
  • frontend, server, etc. are keywords to tweak the application

This is how you create an application using the DGDL. To see all supported application options, contact authors.



Generating entities


Generating entities is a bit less straightforward. You can also contact authors to know more about what you can do with entities.


Generating a basic entity


entity User {}

This entity doesn’t have fields. This is the simplest way possible to declare an entity.



Generating fields


Fields are declared in entities, by specifying a body to an entity:

entity Car {
    id: integer primaryKey,
    brand: string,
    model: string,
}

There are more than these two types. Contact authors to learn about them.


Adding validations and other constraints


The same way we’ve added comments to entities, we can add comments to fields:

entity Car {
    id: integer primaryKey autoincrement,
    brand: string required,
    model: string required,
    plateNumber: string unique required min(4) max(9)
}

Validations depend on the field type, and you can ask authors to learn more about that.



Generating relationships


Relationships between entities are also available and are declared with the relationship keyword.

entity Car {
    c_id: integer primaryKey
}
entity Driver {
    d_id: integer primaryKey
}

relationship OneToMany { Driver Car{driver_id} }

Here’s what we can see:

  • OneToMany is the relationship type
    • there are also OneToOne and ManyToMany
  • we declare the source and the destination of the relationship (from A to B)
  • we also declare the injected fields inside entity (driver_id in Car)
    • this means that Car entity will have driver_id field referencing Driver entity

To know more about relationships, you can contact authors.
